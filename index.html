Discord.py Music Bot

May 2017


Discord is a popular voice chat application capable of running on most major operating systems. Because of how easily bots can be integrated into the servers, there is a large community devoted to bot development.





Goal: Playing Mp3 Files


After playing around with the Discord API, I decided to make my own music bot. There are many very popular music bots out such as Just-Some-Bots' Music Bot, but I've noticed that most of them use Youtube Downloader instead of mp3 files already on the owner's computer.





Since my computer already has a large collection of mp3 files, I decided to modify the example playlist bot from the Discord.py API. Like Just-Some-Bots, this example also uses Youtube Downloader, but it provided a good start point.


Stream a Single Mp3 File


Playing a single Mp3 file was fairly simple. The create_ffmpeg_player("file.mp3") method from the API creates a Player object and this streams the music once a bot connects to a voice channel. So the "play" command was replaced by a random search for an mp3 from a Music folder and a new player creation.


Continuous Playing


The original code relied on a playlist and a queue of music from Youtube requested by people in the chat. Unfortunately, this needed to be completely scrapped for this Mp3 Shuffle Bot. Once again, the create_ffmpeg_player("file.mp3") had a solution for this. There is an "after" parameter in the method which runs a function immediately after the stream ends. By having it call a cont method to reroll the song and continue playing, I created an unending playlist. Finally, the forward and backward options were recycled from my previous music player project, PyMusic. The forked version of Discord.py API with my new music bot can be found here.



Movie Tracker Java

March 2017


It's spring break and time for a new project! This time I'll be working on a java application to track movie ratings on IMDb. First off, I'll need to web scrape information from their site.


Web Scraping with Jsoup


Luckily, I found Jsoup pretty quickly. It's an open source java library for interpreting html on webpages. It was easy to use and very similar to the module Beautiful Soup that I used for webscraping with python. With the webscraper all set, the next step will be interpreting the information.


Data Structures


To store all of this information, I created a Movie Class. This could record everything from titles, directors, and ratings. The information from the html was relatively easy to parse, so everything could be sent in through the constructor for the class. There is even room to add personal ratings to each movie.


Up next is the issue of being able to sort the movies by rating. First, I set up an Arraylist to store all of the movies and tried a heap sort. Although it worked fine for sorting the movies, I eventually settled on a quicksort for a cleaner implementation.


Future Goals


The basis for the Movie Ratings Java App have been set up. There are still a lot of directions this project can go. I will be planning on setting up a GUI and extend the movie scraper to more than IMDb Top 250.



Minimalistic Python Music Player Overlay

Jan 2017

I'm always a fan of minimalistic design. The goal for this project was to create a lightweight Music Player with only 6 buttons, Play, Pause, Forward, Backward, and Volume Controls. Similar to a Digital iPod Shuffle."



iPod Shuffle

I tried to start with the python vlc module, but it was a bit too reliant on plugins. The next best option was to use a self-sufficient module that could play mp3 files. Luckily I found pygame. Then after adding a few basic data structures to the code (Queue, Stack, etc), a skip and rewind function were created.


Graphical User Interface


With the music player organized, it was time to make the GUI. The Tkinter python module made it very simple.

1. Create a window
2. Add in the buttons along a grid
3. Make a draggable label
4. Use overrideredirect

The result was an unobstrusive music player overlay and with Pyinstaller I converted this into an executable file. I was very happy with how it ended up.

Travel Helper: Check Gas Prices

Jan 2017

During the Spring and Summer, I do a lot of hiking. Unfortunately, most of the nice trails are over an hour away from me, which means I'm not very familiar with the area.

I remember one of my trips during this past Summer ended in a mess. We nearly ran out of gas somewhere in Fredrick County, so we grabbed the first available gas station and filled our tank. However, every gas station we saw afterwards was significantly cheaper and I felt we were a bit ripped off.

So, I'll be writing a Python bot to find the lowest gas prices in a small radius. The original plan was to use Google Places API and request the prices for gas_stations, but I ran into a little problem


Google Places API JSON response


As you can see, the API request can get location just fine, but there's no listing for prices. It's really strange because the prices for each gas station are still listed on google maps.


myGasFeed API JSON response


With Google Places not cooperating, I took a look at other API's and myGasFeed looked promising. The API was relatively easy to implement and the bot seems successful. It will need to be approved, but once it is finished you will be able to send a location from the FB mobile app and then ask "What gas stations are near me?" to get a list of all gas stations within a 2 mile radius ordered by distance, their prices, and their address. All you had to send was "http://api.mygasfeed.com/stations/radius/(Latitude)/(Longitude)/(distance)/(fuel type)/(sort by)/(APIKEY).json?"


Final Product


The code was relatively simple, just send the url request with the key and gps coordinates, then parse the json response. The errors I had were mostly typos and they were cleared up with a few print statements and a bit of log monitoring. The most embarassing error was an extra "\n" in the default location text file. I didn't catch it until one of the Facebook tester accidentally hit an infinite loop. Luckily, I shut the app down and repaired it before that went out of control.
